Help generating star fields

Welcome to the starfield generator by Martin Stellinga. This tool can help you generate and save images of star fields.

The user interface consists of two sections:

  • The top section containing the settings.
  • The bottom section containing the generated image.

In the top section, you can create the various layers that will make up the star field you want to generated. There are two types of layers: star layers and nebula layers. Before going into the settings for each of these, it is important to understand how to create such layers.

Creating, deleting and generating layers

When you first open the application, three example layers will be present. They are labeled 'A', 'B', and 'C', respectively.

To create the actual image, the settings have to be generated into an actual star fields. That can be done using the generate button to the right for each individual layer, or you can generate all of them using the render all button at the bottom of the settings section.

Note that the 'generate' buttons always regenerate their layer, while the 'render all' only generates layers if required because of setting changes.

Next to the 'generate' buttons are 'X' (delete) buttons, which can be used to remove a layer.

To add a layer, use the add button in the lower section of the screen. This puts the image section into 'add mode' where you can click on the image to add a new layer with the first coordinate at the place you click. Click done to return to your image.

Coordinates

As noted above, all layers have one or more coordinates that determine where they randomly generate stars or nebulae. You can manipulate these coordinates in move mode.

If you click on Move in the lower section, you will go to 'move mode'. In that mode you can move coordinates of layers around. All coordinates are marked with the letter of the section, followed by the point index. Click done to return to your image.

You can also edit the coordinates in the settings section. If you click the 'V' button of a layer, the settings will expand and reveal a numbered list of coordinates. For each of these coordinates you can edit the X and Y values. Note that Y goes from top to bottom, meaning y = 0 is at the top of the image. X goes from left to right.

To hide the coordinates again, click the '^' button.

Star Layers

A star layer generates a (definable) number of stars in random places of the image. You can adjust the following settings for those generated stars:

  • Layer type:If you change this to 'Nebula', the layer changes to a nebula layer (see below).
  • Star type: This can be 'Circle', 'Rect', or 'Path'. A Circle layer has a single coordinate and generates stars in a circle around that coordinate. A Rect layer has two coordinate and generates stars in a rectangle with the left-top corner and bottom-right corner at these two coordinates. A Path layer has as many coordinates as you want. It generates stars in a radius around the path formed by the coordinates of this layer.
  • Gen. radius:These two values determine the radius for randomly generating stars. The randomizer favors the first radius over the last radius. Setting the first value lower than the second will cluster stars in the inner radius. If you set them to equal values, the stars will be more evenly distributed in the circle/path.
  • #The number of stars to randomly generate in this layer.
  • Brightness:Determine the brightness of the generated stars. The brightness of individual stars is distributed evenly, however you can use the slider to change the brightness value for all generated stars in the layer.
    If you already generated this layer, when you release the slider, the image is automatically updated with the new brightness.
  • Bloom:Determine how much blooming happens to the stars in this layer. A lower value will result in pinprick stars, where a high value will result in the star blooming out.
    If you already generated this layer, when you release the slider, the image is automatically updated with the new brightness.

Nebula Layers

A nebula layer generates a (definable) number of nebulae in random places of the image. You can adjust the following settings for those generated nebulae:

  • Layer type:If you change this to 'Stars', the layer changes to a star layer (see above).
  • Nebula type: This can be 'Circle' or 'Path'. A Circle layer has a single coordinate and generates nebulae in a circle around that coordinate. A Path layer has as many coordinates as you want. It generates nebulae in a radius around the path formed by the coordinates of this layer.
  • Gen. radius:These two values determine the radius for randomly generating nebulae. The randomizer favors the first radius over the last radius. Setting the first value lower than the second will cluster nebulae in the inner radius. If you set them to equal values, the nebulae will be more evenly distributed in the circle/path.
  • # seeds:The number of nebulae to randomly generate in this layer. Note that each nebulae starts with a randomly generated seed (which the following settings affect) which is then expanded using a fractal algorithm.
    'Fractal' means the initial seed (a circle) is subdivided a number of times to create a random cloud.
  • Seed radius:The radius of the intial nebula seeds. The size of the seeds is randomly chosen between these two values.
  • # fractals:The number of times to subdivide the initial nebula seed.
  • Shrink/step:The amount to reduce the radius by at each subdivision. Values above 0.5 will result in a cloud-like nebula, while values below 0.5 will result in isolated puffs.
  • Divisions/step:How many new circles to generated at each subdivision step. The circles are evenly created with the center along the rim of their parent.
  • Angle shift/step:When subdividing the nebula, the children are evenly distributed along the rim of their parents, but the initial angle will be shifted by the value (in radians) in this setting.
  • Hue:The color of this nebula.
    If you already generated this layer, when you release the slider, the image is automatically updated with the new hue.
  • Fade:For each of the subdivided circles of the nebula, the fade determine if they should fade at the center (inner) or outside (outside).
    If you already generated this layer, when you release the slider, the image is automatically updated with the new brightness.
  • Brightness:Determine the brightness of the generated nebulae. Note that nebulae are lighted partially by the stars and partially by this value.
    If you already generated this layer, when you release the slider, the image is automatically updated with the new brightness.
  • Smooth:Determine how much smoothing happens to the nebula in this layer. Higher settings will reduce contrasting edges.
    If you already generated this layer, when you release the slider, the image is automatically updated with the new brightness.

General settings

Below the layers are some general settings that affect the rendering:

  • Gas fog effect:This slider effects how the stars and nebulae interact. Higher values will cause more of a blooming effect for stars in nebulae.
    When you release the slider, the image is automatically updated with the new brightness for all layers that were already generated.
  • Image size: Which, unsurprisingly, changes the size of the image to generate.
